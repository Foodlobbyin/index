# Roadmap

This document describes what has been completed, what needs attention, and what is planned next.

For the current project status summary, see [PROJECT_OVERVIEW_AND_STATUS.md](./PROJECT_OVERVIEW_AND_STATUS.md).

---

## Completed

These features are built and working:

- **Incident handling** — Users can report and track vendor incidents.
- **Evidence tracking** — Supporting documents and evidence can be attached to incidents.
- **Moderation features** — Moderators can review, approve, and act on reported incidents.

---

## Needs attention

These areas exist in the codebase but require further work before they are reliable:

- **Reputation system** — The foundation is in place, but scoring logic and display are incomplete.
- **Audit log search history** — Audit logs are recorded but cannot yet be searched or filtered by users.

---

## Upcoming

Once the items above are addressed, the following enhancements are planned:

- **Enhanced reputation system** — Refine scoring, add weighting for incident severity, and display reputation scores on company profiles.
- **Search history for audit logs** — Add a search and filter interface so administrators can query the audit log efficiently.

---

## Next steps

1. Finalize testing of completed features (incident handling, evidence tracking, moderation).
2. Implement the reputation system scoring logic.
3. Build the audit log search interface.
4. Review and update this roadmap after each milestone.

For a practical guide on what to work on based on your available time, see [NEXT_STEPS.md](./NEXT_STEPS.md).
